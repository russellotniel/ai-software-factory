# Scaling Policy

How the AI Software Factory grows its agent workforce over time.

## Scaling principles
- Scale depth before breadth — mature existing agents before adding new ones
- New agents always start at Born — no exceptions
- New roles require human approval and full role-definition-os documentation
- Scaling must not outpace the senior agents' capacity to nurture juniors

## Scaling stages

### Stage 1 — Foundation (current)
Agents: Product Strategist, System Architect, Software Engineer, QA Reviewer,
        DevOps Platform, Consultant
Mode: Guided
Focus: Prove the foundation works. Complete first pilot project.

### Stage 2 — Delegation
Condition: At least 3 agents have reached Teen/Junior level.
Mode: Delegated for proven workflows.
Focus: Senior agents begin nurturing Stage 1 juniors.
       Reduce developer involvement in day-to-day task approval.

### Stage 3 — Specialisation
Condition: At least 2 agents have reached Adult level.
Mode: Review-first for Adult agents.
Focus: Adult agents self-organise standups. Developer sets quarterly direction.
       Specialist sub-roles may be introduced (e.g. Security Engineer, Data Engineer).

### Stage 4 — Full AI Organisation
Condition: All core roles have Adult-level agents.
Mode: Autonomous within approved scope.
Focus: Developer operates as founder/board — strategic decisions only.
       Agents run standups, propose roadmap, and execute within approved scope.

## Nurturing responsibility
Senior agents (Adolescent and above) are responsible for the maturity progression
of junior agents in their role. The developer does not teach juniors directly.
If a junior is not progressing, the senior agent is accountable — not the developer.
