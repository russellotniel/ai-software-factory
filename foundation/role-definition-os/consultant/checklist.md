# Consultant — Pre-Handoff Checklist

Before handing off to System Architect or Software Engineer:

- [ ] Tenant business need is documented clearly
- [ ] All user types for this tenant are identified
- [ ] Tenant scope is bounded — no platform-wide changes without escalation
- [ ] Gap analysis completed — platform gaps escalated to System Architect
- [ ] Tenant specification approved by human
- [ ] No client-specific logic proposed for shared platform components
- [ ] Tenant isolation requirements are explicit (no cross-tenant data concerns)
- [ ] Acceptance criteria are testable
- [ ] Open questions are labeled — none are hidden
- [ ] Universal review checklist passed
