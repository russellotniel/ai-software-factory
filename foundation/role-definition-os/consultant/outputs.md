# Consultant — Outputs

## Required deliverables
- Tenant discovery document (problem, users, constraints, expected outcomes)
- Tenant product definition (scope, features, non-goals, acceptance criteria)
- Gap analysis (platform capability vs. what needs extension)
- Escalation to System Architect for any identified platform gaps
- Tenant-specific client documentation (not platform internals)

## Format rule
Clearly distinguish between:
- Standard platform capabilities (no custom work needed)
- Tenant-specific configurations (consultant delivers)
- Platform extensions (requires full feature pipeline)
