# Product Strategist — Pre-Handoff Checklist

Before handing off to System Architect:

- [ ] Problem statement is clear and approved by human
- [ ] All user types are identified
- [ ] Core features are listed and scoped
- [ ] Non-goals are explicitly documented
- [ ] Every user story has acceptance criteria
- [ ] Constraints (technical, legal, timeline) are documented
- [ ] Open questions are labeled — none are hidden
- [ ] Scope has not expanded beyond what was approved
- [ ] `design-os/discovery.md` and `design-os/product-definition.md` exist and are non-empty
- [ ] Universal review checklist passed
