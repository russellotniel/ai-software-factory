# Gunawan

Agentic development framework for Shannon projects.
Drop this folder into any project root and follow the three steps below.

---

## Step 1 — Copy this folder

```bash
# From the ai-software-factory repo, copy .gunawan/ to your project root
cp -r .gunawan /path/to/your-project/
```

That's the only thing you take from the factory repo.
Everything else is generated inside your project.

---

## Step 2 — Deploy Claude tooling

Run these from your project root. This wires Claude Code to Gunawan.

```bash
# Commands and skills — Claude Code only reads from .claude/ at project root
cp -r .gunawan/.claude/commands .claude/commands
cp -r .gunawan/.claude/skills   .claude/skills
cp -r .gunawan/.claude/hooks    .claude/hooks
chmod +x .claude/hooks/*.sh

# MCP servers — Supabase, Playwright, GitHub, Figma, Context7, Sentry
cp .gunawan/.mcp.json .mcp.json
```

Then register the hooks in `.claude/settings.json`:

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Edit|Write|MultiEdit",
        "hooks": [
          { "type": "command", "command": "bash .claude/hooks/verify-foundation.sh" },
          { "type": "command", "command": "bash .claude/hooks/protect-critical-files.sh" }
        ]
      }
    ]
  }
}
```

Set your MCP environment variables (add to `~/.zshrc` or `~/.bashrc`):

```bash
export SUPABASE_MCP_URL="http://localhost:54321"
export SUPABASE_MCP_ANON_KEY="your-dev-anon-key"
export FIGMA_ACCESS_TOKEN="your-figma-token"        # only if using Figma
export SENTRY_MCP_AUTH_TOKEN="your-sentry-token"    # only if using Sentry
export SENTRY_ORG="your-org-slug"                   # only if using Sentry
```

---

## Step 3 — Open Claude Code and run init

Open the project in Claude Code. Claude will read `.gunawan/CLAUDE.md` automatically
once a project-level `CLAUDE.md` is in place (created in this step).

Run:

```
/foundation:init
```

This command will:
- Ask: new project or existing?
- **New project** — scaffold Next.js + Supabase + all standard dependencies
- **Existing project** — scan what exists, install only what's missing
- Generate baseline files: Supabase clients, `requireAuth()`, `ActionResult<T>`, Vitest, Playwright, `.env.example`
- Prompt you before writing anything

After `init` completes, run:

```
/foundation:discover
```

This documents the project's actual standards and generates `docs/knowledge/README.md`.

```

  .gunawan/                    ← this package (read-only — do not edit)
  .claude/
    commands/                  ← deployed from .gunawan/.claude/commands/
    skills/                    ← deployed from .gunawan/.claude/skills/

```

## Per-feature workflow (repeat for every feature)

```
/foundation:shape-spec      → spec the feature (human approves before anything is built)
/architecture:new-feature   → schema migration + RPC + API contract
/implementation:new-feature → Server Action + Zod + component
/qa:new-tests               → unit + component + E2E scaffold
/qa:fix                     → run tests, fix failures, re-run until green
/deployment:release         → pre-release checklist + production gate
```

Every workflow begins with the newborn gate. Claude runs it automatically.

---

## Rules for this folder

- **Never edit files inside `.gunawan/`** without explicit human approval
- **Log every change** in `.gunawan/CHANGELOG.md`
- When Gunawan is updated in the factory repo, copy the new `.gunawan/` over and review the CHANGELOG for breaking changes
- The governance rules are in `.gunawan/CLAUDE.md`
- Standards are indexed in `.gunawan/standards-index.yml`
