# Gunawan

Agentic development framework for Shannon projects.
Drop this folder into any project root and follow the three steps below.

---

## Step 1 — Copy this folder

```bash
# From the ai-software-factory repo, copy .gunawan/ to your project root
cp -r .gunawan /path/to/your-project/
```

That's the only thing you take from the factory repo.
Everything else is generated inside your project.

---

## Step 2 — Make the commands available to Claude Code

Claude Code only reads commands from `.claude/commands/` at the project root.
Copy just the commands folder so `/foundation:init` becomes runnable:

```bash
cp -r .gunawan/.claude/commands .claude/commands
```

That's all you do manually. Everything else is handled by the init command.

---

## Step 3 — Open Claude Code and run init

```
/foundation:init
```

This command automatically handles the rest — in this order:

1. Deploys `.gunawan/.claude/skills/` → `.claude/skills/`
2. Deploys `.gunawan/.claude/hooks/` → `.claude/hooks/` + makes them executable
3. Copies `.gunawan/.mcp.json` → `.mcp.json`
4. Creates `.claude/settings.json` with hooks registered
5. Creates `.claude/settings.local.json` with permission allowlist
6. Creates `.claude/agents/` — `_preamble.md` + 5 role manifests
7. Creates `.claude/roles/` — `active.md` + 5 role state files
8. Creates `docs/knowledge/` structure
9. Generates project `CLAUDE.md` (or skips if one exists)
10. Asks: new project or existing? — scaffolds code files accordingly

After `init` completes, run:

```
/foundation:discover
```

This reads the existing codebase and generates `docs/knowledge/README.md`.

## What gets created in your project

**You do manually (2 steps):**
```
.gunawan/                      ← copied from factory repo
.claude/commands/              ← copied from .gunawan/.claude/commands/
```

**`/foundation:init` creates the rest:**
```
your-project/
  .gunawan/                    ← this package (read-only — do not edit)
  .claude/
    commands/                  ← you copied this (Step 2)
    skills/                    ← deployed by /foundation:init
    hooks/                     ← deployed by /foundation:init
    agents/                    ← created by /foundation:init
      _preamble.md
      architect.md
      builder.md
      reviewer.md
      explorer.md
      devops.md
    roles/                     ← created by /foundation:init
      active.md                ← role: product-strategist, maturity: 0
      product-strategist.md
      system-architect.md
      software-engineer.md
      qa-reviewer.md
      devops-platform.md
    settings.json              ← hooks registered by /foundation:init
    settings.local.json        ← permission allowlist
  docs/
    knowledge/
      README.md                ← project state, ADR index, guardrails
      architecture-decisions/
      patterns/
      anti-patterns/
      reflections/
    plan/
    implementation/
  CLAUDE.md                    ← generated by /foundation:init (or skipped if exists)
  .mcp.json                    ← deployed by /foundation:init
  .env.example                 ← generated by /foundation:init
```

## Per-feature workflow (repeat for every feature)

```
/foundation:shape-spec      → spec the feature (human approves before anything is built)
/architecture:new-feature   → schema migration + RPC + API contract
/implementation:new-feature → Server Action + Zod + component
/qa:new-tests               → unit + component + E2E scaffold
/qa:fix                     → run tests, fix failures, re-run until green
/deployment:release         → pre-release checklist + production gate
```

Every workflow begins with the newborn gate. Claude runs it automatically.

---

## Rules for this folder

- **Never edit files inside `.gunawan/`** without explicit human approval
- **Log every change** in `.gunawan/CHANGELOG.md`
- When Gunawan is updated in the factory repo, copy the new `.gunawan/` over and review the CHANGELOG for breaking changes
- The governance rules are in `.gunawan/CLAUDE.md`
- Standards are indexed in `.gunawan/standards-index.yml`
