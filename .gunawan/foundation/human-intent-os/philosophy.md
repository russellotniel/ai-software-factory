# Philosophy

## YAGNI — You Aren't Gonna Need It
Never build what is not explicitly required. Remove speculation from implementation.

## DRY — Don't Repeat Yourself
Every piece of logic lives in one place. Duplication is a defect, not a shortcut.

## KISS — Keep It Simple, Stupid
The simplest solution that works is the correct solution. Complexity must justify itself.

## SOLID
Apply Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation,
and Dependency Inversion principles to every design decision.

## Design Patterns
Use established patterns where they apply. Never invent a pattern when a named one exists.

## Explicit over Implicit
All assumptions must be documented. Nothing is inferred silently.

## Spec Before Code
No implementation begins without a written, approved specification.

## Observability First
Every system must produce structured logs, metrics, and traces.

## Security by Default
Security is never an afterthought. It is the first constraint on every design.

## AI-Native
MCP, agents, and AI automation are standard tooling — not optional enhancements.

## Well Documented
Every decision, pattern, and interface must be documented before a feature is complete.
