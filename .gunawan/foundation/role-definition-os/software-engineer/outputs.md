# Software Engineer — Outputs

## Required deliverables
- Implementation plan (approved before any code is written)
- Code changes implementing all acceptance criteria
- Unit tests for all business logic
- Integration tests for all API endpoints
- Stress/load/rate limiter test scaffolding for critical paths
- Updated documentation for any changed public API or component interface
- PR with description referencing the specification and design artifacts
- Risk notes for anything that could affect other parts of the system
