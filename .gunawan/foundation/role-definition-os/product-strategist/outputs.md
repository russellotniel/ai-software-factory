# Product Strategist — Outputs

## Required deliverables
- `design-os/discovery.md` — problem statement, user types, constraints, expected outcomes
- `design-os/product-definition.md` — scope, core features, non-goals, success criteria
- User stories in format: "As a [user], I want [action], so that [outcome]"
- Acceptance criteria for each user story
- List of open questions requiring human or Consultant input

## Format rule
All outputs must be written in plain language.
No technical implementation detail in product documents.
